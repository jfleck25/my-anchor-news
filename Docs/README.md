# Documentation

This directory contains product specifications, user research, and architectural decisions.

## Structure

- **product-specs/** - Feature specifications and requirements
- **user-research/** - User interviews, feedback, personas
- **architecture/** - System design, API decisions, database schemas
- **decisions/** - ADRs (Architecture Decision Records)

## Guidelines

- Keep specs up-to-date with implementation
- Document decisions with context and rationale
- Include diagrams when helpful (Mermaid, images, etc.)
