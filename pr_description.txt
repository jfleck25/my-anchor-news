🎯 **What:**
Fixed a code health issue where `pool` was imported but unused in `main.py` (Flake8 F401 warning).

💡 **Why:**
The line `from psycopg2 import pool` imported `pool` into the global namespace but it was never used directly. Instead, `psycopg2.pool.ThreadedConnectionPool` was used. While this syntax implicitly made the `psycopg2.pool` submodule available, it triggered linter warnings for an unused import.
Replaced `from psycopg2 import pool` with `import psycopg2.pool`, making the required submodule available for `ThreadedConnectionPool` while correctly silencing the unused import warning. Also explicitly mocked `psycopg2.pool` in `test_main.py` since it's an explicit top-level import now.

✅ **Verification:**
- Ran `flake8 main.py` and confirmed that the F401 unused import warning has been resolved.
- Ran tests suite for tests importing `main.py` with mock, and verified that Pytest collections run successfully.

✨ **Result:**
Cleaner code that adheres to flake8 style guidelines without triggering spurious unused import warnings or breaking the database connection pool logic.
