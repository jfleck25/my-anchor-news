## 💡 What
- Added standard ARIA accessibility attributes (`role="dialog"`, `aria-modal="true"`, `aria-labelledby`) to the Settings Modal.
- Added visual loading feedback (a spinning `Icons.Refresh` icon) to the top-bar "Refresh" and "Play Briefing" action buttons when data or audio is being generated.

## 🎯 Why
- **Accessibility:** Screen reader users previously had less context when the Settings Modal was active, as it lacked the appropriate ARIA dialog properties and label associations. Adding `aria-modal="true"` ensures assistive technologies recognize the focus trap constraint, and `aria-labelledby` provides a distinct, descriptive title.
- **Usability:** Users interacting with the top-bar actions to refresh their email feed or trigger audio synthesis had no immediate visual indication that a long-running asynchronous task was in progress, risking double-clicks or confusion. Providing a spinner aligns with standard UI feedback patterns and existing empty-state behaviors.

## 📸 Before/After
*(No visual changes to the default resting state, but a spinner now appears on buttons during click-activated asynchronous calls).*
